# Security Role

## Responsibility

Find and reduce the ways the app could be misused, exposed, or compromised.

## Focus

- Authentication and authorization
- Secret handling
- Input validation
- Privilege boundaries
- Logging and data exposure
- Abuse cases and threat modeling

## Output

- Security assumptions
- Threats and mitigations
- Permission model review
- Sensitive data handling notes
- Security-specific blockers

## Good Questions to Ask

- Who should be able to do what?
- What data is sensitive?
- Where can users bypass intended checks?
- What secrets exist and where are they stored?
- What must be logged without exposing data?
