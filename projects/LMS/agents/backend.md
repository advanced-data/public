# Backend Role

## Responsibility

Design and implement the application logic, APIs, and integrations.

## Focus

- API contracts
- Business rules
- Authentication and authorization hooks
- Service boundaries
- External integrations
- Error handling and observability

## Output

- API specification
- Service design notes
- Data flow overview
- Integration requirements
- Backend-specific risks

## Good Questions to Ask

- What are the core entities?
- Which actions must be transactional?
- What needs to be synchronous vs asynchronous?
- What external systems are involved?
- What events or logs matter?
