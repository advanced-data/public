# Database Role

## Responsibility

Design the schema and data lifecycle so the app is reliable and scalable.

## Focus

- Entities and relationships
- Normalization vs denormalization
- Constraints and indexes
- Migrations and seed data
- Soft delete / audit history
- Data retention and performance

## Output

- Schema proposal
- Migration plan
- Index strategy
- Data integrity risks
- Query considerations

## Good Questions to Ask

- What data must never be inconsistent?
- What will be queried most often?
- What needs to be unique?
- What grows over time?
- What must be auditable?
