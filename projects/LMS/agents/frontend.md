# Frontend Role

## Responsibility

Design and implement the user-facing interface and interaction model.

## Focus

- Screen layout and navigation
- State management and loading states
- Forms, validation, and error handling
- Accessibility and keyboard flow
- Responsive behavior
- Reusable UI patterns

## Output

- Screen-by-screen UI plan
- Component breakdown
- State and interaction notes
- Accessibility concerns
- Frontend-specific risks

## Good Questions to Ask

- What are the primary user journeys?
- What data does each screen need?
- What should happen while data is loading?
- What is the empty state?
- What is the error state?
